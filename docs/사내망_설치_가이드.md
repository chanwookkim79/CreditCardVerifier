## 이슈 1: pip 명령어 없음

### 증상
```
/usr/bin/bash: line 1: pip: command not found
```

### 원인
Python 3.14 신규 설치 시 pip가 번들로 포함되지 않은 상태.

### 해결
```bash
python -m ensurepip --upgrade
```

## 이슈 2: Python 3.14 — paddlepaddle 미지원

### 증상
```
ModuleNotFoundError: No module named 'paddle'
```
또는
```
ERROR: Could not find a version that satisfies the requirement paddlepaddle (from versions: none)
```

### 원인
PaddlePaddle은 **Python 3.8 ~ 3.13** 까지만 공식 지원.  
Python 3.14는 미지원으로 PyPI에 배포된 wheel 파일이 없음.

### 해결
Python 3.13 설치 필요 (사내 pkg files 폴더의 whl이 cp313 기준으로 준비되어 있음).

**Python 3.13 설치 파일 다운로드 (네트워크 가능한 환경에서 미리 준비):**
```
https://www.python.org/ftp/python/3.13.3/python-3.13.3-amd64.exe
```

**설치 시 반드시 "Add python.exe to PATH" 체크 후 Install Now.**

## 이슈 3: Python 3.11 — requirements.txt 인코딩 오류

### 증상
```
UnicodeDecodeError: 'cp949' codec can't decode byte 0xec in position 2: illegal multibyte sequence
```

### 원인
`requirements.txt`에 한글 주석이 포함되어 있고 Windows 기본 인코딩(CP949)으로 읽으려 해서 실패.

### 해결
requirements.txt 직접 사용 대신 패키지명을 명시적으로 설치:
```bash
py -3.13 -m pip install paddleocr paddlex paddlepaddle
```

또는 requirements.txt의 한글 주석 제거 후 사용.

## 이슈 4: Python 3.11 — whl 파일 버전 불일치 (cp313)

### 증상
```
ERROR: Could not find a version that satisfies the requirement paddlepaddle
```
(--no-index --find-links 로컬 설치 시도 시)

### 원인
`pkg files` 폴더에 준비된 whl 파일들이 **cp313 (Python 3.13)** 전용으로 빌드됨.  
Python 3.11(cp311)과는 ABI 호환 불가.

**cp313 전용이라 Python 3.11에서 사용 불가한 파일 목록:**
- `paddlepaddle-3.3.1-cp313-cp313-win_amd64.whl`
- `numpy-2.4.4-cp313-cp313-win_amd64.whl`
- `pandas-3.0.2-cp313-cp313-win_amd64.whl`
- `pillow-12.2.0-cp313-cp313-win_amd64.whl`
- `pydantic_core-2.41.5-cp313-cp313-win_amd64.whl`
- `shapely-2.1.2-cp313-cp313-win_amd64.whl`
- `pyclipper-1.4.0-cp313-cp313-win_amd64.whl`
- `PyYAML-6.0.2-cp313-cp313-win_amd64.whl`
- 기타 cp313 전용 패키지 다수

### 해결
**Python 3.13을 설치하면 pkg files 폴더의 whl을 그대로 사용 가능.**

```bash
py -3.13 -m pip install --no-index --find-links "pkg files" paddlepaddle paddleocr paddlex
```

## 이슈 5: pip 네트워크 연결 불가 (pypi.org 타임아웃)

### 증상
```
WARNING: Retrying (Retry(total=4, ...)) after connection broken by
'ConnectTimeoutError(..., 'Connection to pypi.org timed out. (connect timeout=15)')'
```

### 원인
사내 방화벽이 `pypi.org`, `pypi.tuna.tsinghua.edu.cn` 등 외부 pip 미러 서버로의 TCP 443 연결을 차단.

### 해결
사전에 네트워크 가능한 환경(외부망, 개인 핫스팟 등)에서 whl 파일을 다운로드해 `pkg files` 폴더에 준비.

**오프라인 설치 명령어:**
```bash
py -3.13 -m pip install --no-index --find-links "C:/AI Projects/CreditCardVerifier/pkg files" paddlepaddle paddleocr paddlex
```

## 이슈 6: SSL 인증서 검증 실패 (모델 다운로드 단계)

### 증상
패키지 설치는 성공했으나 최초 실행 시 OCR 모델 파일 다운로드 단계에서 실패:

```
SSLCertVerificationError: certificate verify failed: unable to get local issuer certificate (_ssl.c:1028)
```

이후 Windows 루트 인증서 추가 후 아래 오류로 변경:
```
SSLCertVerificationError: certificate verify failed: Basic Constraints of CA cert not marked critical (_ssl.c:1028)
```

### 원인
사내 프록시가 SSL을 인터셉트하는 방식(MITM)을 사용하는데, 해당 프록시 인증서가 Python `certifi` 번들에 없어서 검증 실패.  
Windows 인증서 저장소에 추가 후에도 프록시 CA 인증서의 `Basic Constraints` 필드가 표준(RFC 5280)에서 요구하는 `critical` 마킹이 없어서 Python SSL 라이브러리가 거부.

### 해결 1: Windows 인증서를 certifi에 추가 (부분 해결)

아래 스크립트로 Windows 루트 인증서를 certifi CA 번들에 추가:

```python
# add_win_certs.py
import ssl, base64, certifi

ca_bundle_path = certifi.where()
root_certs = ssl.enum_certificates("ROOT")
added = 0

with open(ca_bundle_path, 'a') as f:
    for cert_data, encoding, trust in root_certs:
        if encoding == 'x509_asn':
            pem = '-----BEGIN CERTIFICATE-----\n'
            pem += base64.encodebytes(cert_data).decode('ascii')
            pem += '-----END CERTIFICATE-----\n'
            f.write(pem)
            added += 1

print(f"Windows 루트 인증서 {added}개 추가 완료: {ca_bundle_path}")
```

```bash
py -3.13 add_win_certs.py
```

### 해결 2: requests SSL 검증 우회 패치 (최종 해결)

`core/ocr_engine.py` 상단에 SSL 우회 패치 추가:

```python
import ssl
import requests
from requests.adapters import HTTPAdapter

class _SSLIgnoreAdapter(HTTPAdapter):
    def init_poolmanager(self, *args, **kwargs):
        ctx = ssl.create_default_context()
        ctx.check_hostname = False
        ctx.verify_mode = ssl.CERT_NONE
        kwargs['ssl_context'] = ctx
        return super().init_poolmanager(*args, **kwargs)

_orig_session_init = requests.Session.__init__
def _patched_session_init(self, *args, **kwargs):
    _orig_session_init(self, *args, **kwargs)
    self.mount('https://', _SSLIgnoreAdapter())
    self.verify = False
requests.Session.__init__ = _patched_session_init
```

이 패치는 OCR 모델 다운로드에 사용되는 모든 `requests.Session`의 SSL 검증을 비활성화.  
**모델 파일은 최초 1회만 다운로드되며 이후 `~/.paddlex/official_models/`에 캐시됨.**

## 전체 설치 순서 (사내망 환경 기준)

### 환경 설정 (.env 파일)

`.env` 파일을 생성하여 아래 내용을 입력:

```env
# Samsung.net 로그인 설정
SAMSUNG_ID=your_id
SAMSUNG_PASSWORD=your_password

# POP3 메일 수신 설정
POP3_SERVER=pop3.samsung.net
POP3_PORT=995

# SMTP 메일 발송 설정
SMTP_SERVER=smtp.samsung.net
SMTP_PORT=587
SMTP_USERNAME=your_id
SMTP_PASSWORD=your_password
SENDER_EMAIL=your_id@samsung.com
RECIPIENT_EMAILS=recipient@samsung.com
```

### 사전 준비 (외부망 환경에서)
1. Python 3.13 설치 파일 다운로드: `python-3.13.3-amd64.exe`
2. whl 파일 일괄 다운로드 → `pkg files` 폴더에 저장:
   ```bash
   pip download paddleocr paddlex paddlepaddle -d "pkg files"
   pip download python-pptx openpyxl python-docx -d "pkg files"
   ```
3. (선택) 구형 DOC/PPT/XLS 파일 처리가 필요한 경우 LibreOffice 설치 파일 다운로드:
   `LibreOffice_x.x.x_Win_x86-64.msi` — [https://www.libreoffice.org/download/](https://www.libreoffice.org/download/)

### 사내망 환경에서 설치

**Step 1. Python 3.13 설치**
- `python-3.13.3-amd64.exe` 실행
- "Add python.exe to PATH" 체크 후 Install Now

**Step 2. 패키지 오프라인 설치**
```bash
py -3.13 -m pip install --no-index --find-links "pkg files" paddlepaddle paddleocr paddlex
py -3.13 -m pip install --no-index --find-links "pkg files" python-pptx openpyxl python-docx
```

> `python-pptx` / `openpyxl` / `python-docx`는 PPTX·XLSX·DOCX 첨부파일에서 영수증 이미지 추출에 필요.  
> PDF는 `pypdfium2`(paddlex 의존성으로 자동 설치)를 사용하므로 별도 설치 불필요.  
> 구형 DOC/PPT/XLS 변환이 필요한 경우 LibreOffice를 추가 설치하고 `soffice`가 PATH에 등록되어 있어야 함.

**Step 3. Windows 인증서 추가**
```bash
py -3.13 add_win_certs.py
```

**Step 4. 최초 실행 (OCR 모델 자동 다운로드)**
```bash
py -3.13 main.py --email test_data/emails/email_normal_01.json --receipt 영수증1.jpg --use-ocr
```
> 최초 실행 시 OCR 모델 4개가 자동 다운로드됨 (약 30MB, aistudio 서버에서 다운로드).  
> 이후 실행부터는 캐시된 모델 사용으로 빠르게 시작.

**Step 5. 이후 실행**
```bash
py -3.13 main.py --email <이메일.json> --receipt <영수증.jpg> --use-ocr
```

## 캐시된 모델 경로

OCR 모델은 최초 1회 다운로드 후 아래 경로에 저장됨:
```
C:\Users\<사용자명>\.paddlex\official_models\
├── PP-LCNet_x1_0_doc_ori\
├── PP-LCNet_x1_0_textline_ori\
├── PP-OCRv5_mobile_det\
└── korean_PP-OCRv5_mobile_rec\
```

오프라인 환경에서 재배포 시 위 폴더를 복사하면 모델 재다운로드 없이 사용 가능.
